Public Request & Response

Email Processing Pipeline
A hybrid human-AI workflow that combines manual email submission with automatic LLM-based classification and entity extraction. Results are presented to a reviewer who confirms the final action.
User submits incoming email data — sender, subject, body and optional attachment
System task fires automatically and calls an LLM API to classify the email
LLM extracts entities, assigns a category, and generates a suggested reply draft
Reviewer sees all AI output, edits the draft if needed, and selects a final action
API key stored as a hidden field — never visible to end users
Uses the hybrid human-system task pattern. The LLM call is made inside a Groovy action via async.run and results are written back with setData. Demonstrates Petriflow's AI integration capability.

List process has a permanent task with an Add button and an embedded taskRef panel
Clicking Add creates a new Item case and adds its task to the embedded list instantly

Item process uses a system-role task so it stays alive and embedded indefinitely

Submitter fills in vendor, amount, description, category and justification
Legal and Finance review run in parallel — AND-split pattern
